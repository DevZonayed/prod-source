Code Editor: Which editor library should we use?

We need a web-based code editor with syntax highlighting, file tabs, and a file explorer. Three main options exist.

A

Monaco Editor (VS Code's engine)

The exact same editor that powers VS Code. Full IntelliSense, multi-cursor, minimap, find & replace, code folding, bracket matching — everything users expect from VS Code.

Pros

  • Identical to VS Code — users already know it
  • Full TypeScript/JSX support built-in
  • Best autocomplete & error highlighting
  • Used by StackBlitz, CodeSandbox, GitHub.dev

Cons

  • Large bundle (~2-4MB)
  • Can be heavy on low-end devices
B

CodeMirror 6

Modern, lightweight, extensible editor. Used by Replit, Observable, and many modern web apps. Good TypeScript support via extensions.

Pros

  • Lightweight (~500KB)
  • Very fast, even on mobile
  • Highly extensible
  • Better accessibility

Cons

  • Less VS Code-familiar
  • TypeScript support requires extra setup
  • No built-in IntelliSense
C

Shiki (Read-only) + Simple Edit

Not a full editor — just beautiful syntax-highlighted code display with basic edit capabilities. Minimal weight, great for viewing AI-generated code. Users can make small edits but it's not a full IDE.

Pros

  • Tiny bundle (~100KB)
  • Beautiful VS Code-quality highlighting
  • Fastest to load

Cons

  • Not a real editor — limited editing
  • No autocomplete, no error highlighting
  • Would need to build editing features

My recommendation: Option A (Monaco) — You specifically asked for "VS Code in the web." Monaco IS VS Code's editor. It's the industry standard for AI app builders (StackBlitz, CodeSandbox, Bolt.new all use it). The bundle size is acceptable for a dev tool.
