AI + Editor Sync: How should AI file changes appear in the editor?

When the AI writes or modifies files via tools (writeFile, replaceInFile), how should the code editor reflect those changes?

A

Auto-Open & Live Update

When the AI creates or modifies a file, it automatically opens as a tab in the editor and the content updates in real-time. The editor auto-switches to the file being modified. User sees the AI "typing" code live.

Pros

  • User sees exactly what AI is doing
  • Feels magical — like pair programming
  • Builds trust and transparency

Cons

  • Can be distracting if AI edits many files fast
  • Editor may flicker during rapid changes
B

Update Silently, Highlight Changed Files

File contents update in the background. Changed files get a dot indicator on their tab and in the file tree (like unsaved changes in VS Code). User stays on whatever file they were viewing — no auto-switching.

Pros

  • Clear indication of what changed

Cons

  • User might miss important changes
  • Less "wow" factor
C

Smart: Auto-Open First File, Then Dot Indicators

When the AI starts a new generation pass, it auto-opens the first file being modified. Subsequent files in the same pass update silently with dot indicators. This gives the "live coding" feel without overwhelming the user during multi-file edits.

My recommendation: Option A — Auto-open and live update. This is the most impressive UX and what users expect from AI builders. Seeing the AI write code in real-time is the magic moment. We can add a small debounce to prevent flicker during rapid edits.
