File Explorer: How should users navigate project files?

The code editor needs a way to browse and open files. How should this work in the Code tab?

A

Sidebar Tree (VS Code style)

A persistent file tree on the left side of the code editor, just like VS Code's explorer. Collapsible folders, file icons, right-click context menu.

Pros

  • Most familiar to developers
  • Always visible, easy to browse
  • Supports drag-drop, rename, delete

Cons

  • Takes horizontal space (~180px)
  • Can feel cramped on smaller screens
B

Collapsible Sidebar (toggle open/close)

Same tree as Option A, but it can be toggled open/closed with a button. Closed by default to maximize editor space. Opens as an overlay or pushes content.

Pros

  • Maximum editor space when closed
  • Full tree when needed
  • VS Code also supports this pattern

Cons

  • Extra click to browse files
C

Tabs Only (no file tree)

No sidebar at all. Files open as tabs when the AI edits them or when the user searches. A quick-open search (Cmd+P) lets users find files by name. Minimal and clean.

Pros

  • Maximum editor space
  • Clean, minimal UI
  • AI auto-opens relevant files

Cons

  • Hard to discover project structure
  • Users expect a file tree

My recommendation: Option B (Collapsible Sidebar) — gives maximum editor space by default, but the full file tree is one click away. This is how VS Code itself works when you toggle the sidebar. Best balance of space and discoverability.
