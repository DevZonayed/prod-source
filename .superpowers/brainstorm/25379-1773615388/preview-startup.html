Preview Startup: What should the user see before the app is ready?

Currently the iframe shows "localhost refused to connect" while the container boots and installs dependencies. This takes 30-60 seconds. What should happen instead?

The Problem Timeline

Current: Broken experience
T+0s
Page loads
iframe: "refused to connect"
→
T+5s
Container starting
Still "refused to connect"
→
T+30s
npm install running
Still "refused to connect"
→
T+60s
Dev server ready
Finally loads!
A

Scaffold First, Preview Instantly

When user creates a project, immediately scaffold a working Next.js app (pre-built template in the Docker image) before the AI starts modifying code. The preview shows the default Next.js page within seconds, then updates live as the AI writes code.

T+3s
Template copied
Preview: Next.js default page
→
T+5s
AI starts editing
Preview: Hot-reloads live
→
T+30s
AI finished
Preview: Complete app

Pros

  • Instant gratification
  • User sees progress in real-time
  • No "refused to connect" ever

Cons

  • Needs pre-built template in Docker image
  • node_modules must be pre-installed
B

Smart Loading Screen (No iframe until ready)

Don't render the iframe at all until the dev server responds. Instead, show a polished loading screen with real-time status updates: "Creating container... Installing dependencies... Starting dev server..."

T+0s
Loading screen
"Creating environment..."
→
T+30s
Loading screen
"Starting dev server..."
→
T+60s
iframe loads
App appears

Pros

  • No "refused to connect" ever
  • Clean, professional feel
  • Easy to implement

Cons

  • User waits 30-60s seeing a spinner
  • No real-time preview of AI progress
C

Both: Scaffold + Smart Loading Fallback

Best of both worlds. Pre-scaffold the Next.js app in the Docker image so the dev server starts in ~3s. But also have a smart loading overlay that only shows briefly and disappears once the iframe actually loads. If anything goes wrong, the loading screen handles it gracefully.

T+0s
Brief loading
"Preparing..."
→
T+3s
Template ready
Preview: Default page
→
T+5s+
AI editing
Preview: Live updates

Pros

  • Fastest possible preview
  • Graceful fallback
  • Real-time AI progress visible

Cons

  • More work (template + loading)

My recommendation: Option C — Pre-scaffold the template so preview is near-instant, with a smart loading overlay as safety net. This completely eliminates the "refused to connect" problem.
