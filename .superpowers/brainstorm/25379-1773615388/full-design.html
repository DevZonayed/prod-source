4. AI → Editor Sync

When AI writes a file:
  • File opens automatically as a new tab
  • Editor switches to that file
  • Content appears in real-time
  • Green highlight on new/changed lines
  • File tree shows the new file
When AI modifies a file:
  • If tab exists, content updates live
  • If no tab, opens a new one
  • Changed lines highlighted in green
  • Scroll follows the changes
  • Preview hot-reloads simultaneously
User can also edit:
  • Full Monaco editing (type, paste, etc.)
  • Changes save to container filesystem
  • Next.js hot-reloads on save
  • Ctrl+S to save (auto-save optional)
  • User edits persist across sessions

5. Component Architecture

New Components
CodeEditor
Monaco editor wrapper. Manages tabs, content sync with container filesystem, AI write events.
FileExplorer
Tree view sidebar. Reads file list from container via API. Supports expand/collapse, file icons, click-to-open.
EditorTabs
Tab bar for open files. Close tabs, reorder, show modified indicator. Active tab highlighted.
useFileSystem (hook)
Reads/writes files via container API. Lists directory tree. Watches for AI file changes via custom events.
useEditorSync (hook)
Bridges AI tool calls (writeFile, replaceInFile) to Monaco editor. Opens tabs, updates content, highlights changes.
PreviewLoadingOverlay
Smart loading screen. Shows status steps. Hides when iframe loads. Never shows "refused to connect".
Modified Components
RepoWorkspaceShell
Add Code/Preview tab toggle. Integrate CodeEditor. Keep terminal at bottom.
Docker Image
Bake in Next.js template with pre-installed node_modules. Copy to /workspace on container start.
dev-server-manager
Skip npm install when template detected. Faster startup path.
